
Los [LLM](LLM.md) para ser entrenados por primera vez (comunmente llamado preentrenamiento) necesitan ser construidos a partir de grandes muestras de datos con un entrenamiento auto supervisado; es decir, sin intervención de un humano durante esta etapa (ningun tipo de etiqueta al dato hecha por un humano).  De esta forma, el modelo de lenguaje construido cuenta con un conocimiento estadístico del lenguaje en el que fue creado, pero sin la capacidad de hacer tareas específicas de manera eficiente. Es por esto, que los LLM posteriormente al prentrenamiento pasan por un proceso llamado **fine tuning** (postentrenamiento de manera supervisada). 

Prentrenar un modelo con grandes cantidades de data requiere de un costo computacional, de tiempo y ambiental muy grande. Por eso, para obtener LLMs que respondan eficientemente a tareas especificas se usan modelos ya entrenados y se ajustan con finetuning a través de datasets especificamente construidos para eso. Incluso de esta forma se ha demostrado a través de [Benchmark](../Hardware%20-%20Fundamentos%20&%20Definiciones/Benchmark.md) que los modelos responden mejor en precisión. 

No confundir transfer learning con [Transformers](../Transformers/Transformers.md). El **Transformer** es la arquitectura (el "cómo" se procesan los datos en paralelo), mientras que el **Transfer Learning** es el paradigma que nos permite no empezar desde cero, aprovechando el conocimiento general del modelo para especializarlo en ingeniería civil.
